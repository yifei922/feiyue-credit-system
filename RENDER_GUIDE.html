🚀 Render 上线图文指南（菜鸟版）

你的代码已经在 GitHub 上：github.com/yifei922/feiyue-credit-system。下面只在浏览器里点几下，把仓库连到 Render 就能上线。

✅ 全程网页操作，不需要 API Key，不需要命令行。

1打开 Render 控制台

浏览器打开 https://dashboard.render.com 并登录你的账号。

┌─────────────────────────────────────────────┐
│ Render [头像] │
│ │
│ 🟢 绿按钮： + New ← 点它 │
│ │
└─────────────────────────────────────────────┘

2选「Blueprint」一键部署

点 + New 后，在下拉里选 Blueprint。（我们的 render.yaml 已写好构建/启动命令，选它最省事）

+ New ▾
├─ Web Service
├─ Background Worker
├─ Blueprint ← 选这个
└─ Static Site

3连接 GitHub 仓库

首次会让你授权 Render 读取 GitHub（点 Authorize / Install）。授权后在列表里找到仓库：

选择仓库：
☑ yifei922 / feiyue-credit-system ← 勾选
☐ other-repo-1
☐ other-repo-2
如果没看到仓库，点页面上的 Configure 或 Refresh 重新拉取 GitHub 仓库列表。

4确认配置 → 创建

Render 会自动读 render.yaml，你只需核对并点创建：

  • Name：feiyue-credit-system（可改）
  • Runtime / Environment：Node
  • Build Command：cd frontend && npm install && npm run build --mode server && cd ../server && npm install
  • Start Command：cd server && npm start
  • Plan：Free
  • 环境变量 JWT_SECRET：已设为 auto-generate（自动生成）

滚到最下面点 Apply / Create Web Service。

5等部署完成 → 拿到网址

页面会显示构建日志（约 2-4 分钟）。出现绿色 Live 后，顶部有你的网址：

🟢 Live
你的网址： https://feiyue-credit-system.onrender.com
🎉 点这个网址就是你的系统！用 admin / 123456 登录，左侧「学生管理」即可导入导出名单/成绩。
⚠️ 关于绑卡：如果创建服务时 Render 提示要填信用卡，那是 $1 临时验证、不会真实扣费（用于防滥用）。你可以填，也可以改用「本地运行」方式（系统本地已验证可跑）。是否填卡由你决定。
⚠️ 免费层数据会清空：Render 免费服务的磁盘是临时的，重启后 SQLite 数据可能丢失。正式长期使用建议：① 本地运行；② 或升级付费挂持久盘；③ 或把名单/成绩定期导出备份（系统已支持导出）。

📌 上线后建议

  • 去 GitHub → Settings → Developer settings → PAT，把这次的令牌 Revoke 掉（更安全）。
  • 用管理员登录后，第一时间在「学生管理」里导入你们班真实名单。
  • 默认账号密码 123456 太简单，建议改密码或至少别公开。
